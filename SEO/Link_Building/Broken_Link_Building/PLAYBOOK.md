# Broken Link Building Playbook

> **Tactic:** Broken Link Building (Dead Link Reclamation)
> **Difficulty:** Medium | **Scalability:** Medium–High | **Link Quality:** Medium–High
> **Time to First Link:** 1–3 weeks per placement

---

## 1. When to Use This Tactic

Use Broken Link Building when:
- You have (or can create) content that matches a dead/broken resource on a target site
- You need a reason to reach out that goes beyond "please link to me" — offering value first
- You want editorial links from niche resource pages, guides, or curated content hubs
- The client has content assets that could replace broken references (tools, guides, glossaries, studies)
- You want a tactically strong outreach hook that editors genuinely appreciate

**Why it works:** You're helping the editor fix their site while simultaneously earning a link. This creates a value-first dynamic that dramatically improves response rates compared to cold link requests.

**Do NOT use Broken Link Building when:**
- The client has no existing content (you need something to replace the broken link)
- You're not willing to create replacement content if needed
- The niche has few resource pages or outbound links (rare, but some B2B niches are thin)
- You need bulk links quickly — this is methodical, not a high-volume spray tactic

## 2. Prospecting Criteria & Qualification Framework

### Target Page Qualification Criteria
| Criteria | Threshold |
|---|---|
| Domain Rating (Ahrefs) | ≥ 30 |
| Monthly Organic Traffic | ≥ 500/mo |
| Spam Score (Moz) | ≤ 15% |
| Topical Relevance | Same or adjacent niche as client |
| Page contains outbound links | Yes (minimum 5 external links) |
| Broken link found | At least one 404/dead link on the page |
| Replacement content exists (or can be created) | Yes |

### Broken Link Qualification Checklist
Before reaching out, confirm:
- [ ] The broken link is actually returning a 404 (not a 301, 302, or temporarily down)
- [ ] The broken link was linking to content topically relevant to the client's existing content
- [ ] The Wayback Machine snapshot confirms what the broken link was about (so you can match the replacement)
- [ ] The client has an equivalent or better replacement page for this content
- [ ] If no replacement exists, is it feasible to create one? (Is the topic worth it?)
- [ ] The linking page is a resource page, guide, or reference article (not a blog post that happens to have an old link)

### High-Value Broken Link Page Types
- Resource pages ("Best [niche] tools," "Ultimate [topic] guide")
- Curated lists and link directories (legitimate niche-specific ones)
- Educational content with citations (university/non-profit pages = high DR)
- Industry glossaries or "what is X" pages
- Blog posts with "further reading" sections
- Wikipedia pages linking to external sources

### Prospecting Methods

**Method 1: Ahrefs Site Explorer — Broken Links on Competitor Sites**
1. Enter competitor domain in Ahrefs Site Explorer
2. Go to Outgoing Links → Broken Links
3. You'll see every broken outbound link on competitor's site
4. Find ones that match client's content
5. Now contact sites that link to these same dead pages (they have the same broken link)

**Method 2: Ahrefs Content Explorer — Dead Pages with Backlinks**
1. Search for [niche keyword] in Content Explorer
2. Filter: "Only broken" (shows pages returning 404)
3. Sort by referring domains (highest first)
4. These are dead pages that OTHER sites are still linking to
5. Create a replacement page on the client's site, then contact all sites linking to the dead URL

**Method 3: Google Search + Check My Links Chrome Extension**
1. Search: `[niche keyword] + "resources"` or `[niche keyword] + "useful links"`
2. Open resource pages in Chrome
3. Run "Check My Links" extension — highlights all broken links in red
4. Note the broken link URL and the context around it
5. Check Wayback Machine to see what the broken link used to point to

**Method 4: Ahrefs "Best by Links" + 404 Filter**
1. In Ahrefs, use Site Explorer on a high-authority niche site
2. Go to Pages → Best by Links
3. Sort by "404" pages (pages that are now dead but had links)
4. These are prime targets — create replacement content, outreach to referring sites

**Method 5: Screaming Frog Crawl of Target Sites**
1. Set up Screaming Frog to crawl a list of target domains
2. Filter for external links returning 404
3. Export the list
4. Cross-reference against client's content to find replaceable dead links

## 3. Step-by-Step Outreach Process

### Phase 1: Research & Asset Preparation (Days 1–3)

**Step 1: Find Broken Links**
- Use Methods 1–5 above to build a list of broken link opportunities
- For each broken link, record: target page URL, broken link URL, Wayback Machine snapshot URL, linking domain DR and traffic

**Step 2: Verify the Broken Link**
- Visit the broken URL — confirm it returns a genuine 404 (not a parked domain or redirect loop)
- Check Wayback Machine (web.archive.org) to view the last cached version of the broken page
- Confirm: what topic did this page cover? Is the client's content a genuine replacement?

**Step 3: Audit the Client's Content**
- Identify which client page is the best match for the broken content
- If no match exists but the opportunity is strong, create a new page or update existing content
- The replacement content must be at least as good (ideally better) than the original broken resource
- Do NOT claim to be replacing a page if your content is totally different from what was there

**Step 4: Find Editor Contact**
- Check site's About or Contact page for editor email
- Use Hunter.io or Apollo.io for email lookup
- Verify email with NeverBounce before sending
- For university/non-profit pages: look for the webmaster or department contact

**Step 5: Craft Personalized Outreach**
- Reference the specific broken link and page it appears on
- Be genuinely helpful — frame it as "I noticed you have a broken link, I wanted to flag it"
- Offer your replacement content as one option, not the only option
- Keep it conversational and non-salesy

### Phase 2: Outreach (Days 3–10)

**Step 6: Send Initial Email**
- Send to the verified editor/webmaster email
- Reference the exact broken link URL and the page it appears on
- Include a Wayback Machine link so they can verify what the broken page was about
- Suggest your replacement content with a clear, brief reason it's relevant
- Do NOT send more than 30 emails/day from a single domain
- Best send times: Tuesday–Thursday, 9–11 AM recipient's local time

**Step 7: Follow Up (Day 5 and Day 10)**
- Follow-up #1: Brief bump — "Just wanted to make sure this didn't get buried"
- Follow-up #2: Final attempt — "Last follow-up; still happy to help either way"
- Maximum 3 touches per prospect
- Do NOT follow up with a broken link report for the same page twice

**Step 8: Handle Replies**
- Editor fixes the link with your content → Confirm the live link; verify it's do-follow and contextual
- Editor says "thanks, we'll fix it" but doesn't use your content → Follow up after 1 week to confirm and gently re-suggest your content
- Editor declines → Thank them politely and move on; never argue
- No reply after 3 touches → Mark exhausted and move to next prospect

### Phase 3: Verification & Scaling (Days 14–30)

**Step 9: Verify Live Links**
- Check the target page to confirm the broken link was replaced with your content
- Verify: link is do-follow, points to correct URL, uses natural anchor text
- Run the client URL through Ahrefs to confirm the new backlink is indexed
- Log everything in your tracking sheet

**Step 10: Scale via Dead Page Campaigns**
- Identify high-traffic dead pages in the niche (Step 4 above: dead pages with many backlinks)
- Create a comprehensive replacement resource on the client's site
- Use Ahrefs to find all sites that link to the dead URL
- Send broken link outreach to all of them simultaneously
- This is the highest-leverage version of broken link building: one page creation → 20–100+ outreach opportunities

## 4. Outreach Email Templates

### Template A: Single Broken Link Report

**Subject:** Broken link on your [Topic] resources page

Hi [First Name],

I was browsing your resource page on [page topic] and noticed one of the links appears to be broken:

**Broken link:** [broken URL]
**Listed as:** "[Link anchor text]" on [page URL]

Looks like the original page is no longer live — you can see the last version on Wayback Machine here: [Wayback URL]

I noticed we have a resource on [client site] that covers the same topic: [client URL]. It might be a useful replacement for your readers.

Happy to help if you need anything else. Either way, thought you'd want to know about the broken link.

[Your Name]
[Title] | [Website]

---

### Template B: Multiple Broken Links on One Page

**Subject:** A few broken links on your [page topic] page

Hi [First Name],

Came across your [page topic] page while researching [niche] resources and wanted to flag a few broken links:

1. "[Anchor text]" — [broken URL] (404)
2. "[Anchor text]" — [broken URL] (404)
3. "[Anchor text]" — [broken URL] (404)

The Wayback Machine still has cached versions of most of these if you want to find replacements.

On a separate note — we have a resource at [client URL] that covers [topic]. If you're looking for a replacement for any of these, it might fit [#1 or #2 specifically].

Either way, hope this helps!

[Your Name]

---

### Template C: Dead Page with Many Backlinks (Mass Outreach)

**Subject:** "[Dead Page Title]" is no longer live — replacement resource?

Hi [First Name],

I noticed your article at [their article URL] links to [dead URL], which appears to have gone offline.

I recently published a comprehensive replacement on this topic: [client URL]

It covers [brief description: 3 bullet points max of what it covers], which I think picks up where the original left off.

Would it make sense to swap the broken link for this one?

[Your Name]
[Title] | [Website]

---

### Template D: Follow-Up #1

**Subject:** Re: Broken link on your [page] resources page

Hi [First Name],

Just circling back on the broken link I flagged last week — wanted to make sure it didn't get lost.

Happy to point you toward a few other replacement options too if [client URL] isn't the right fit.

[Your Name]

---

### Template E: University / Non-Profit (Higher Trust Required)

**Subject:** Citation link issue on [Department] page

Hello,

I was reviewing resources linked from your [specific page URL] and wanted to flag that one citation appears to be offline:

**Dead link:** [URL]
**Last cached version:** [Wayback Machine URL]

The original resource covered [topic]. We maintain an actively updated version of this resource at [client URL] that might serve as a suitable replacement for students and researchers referencing this page.

I've also noted [1-2 other broken links] on the page, in case those are helpful to flag.

Thank you for maintaining such a valuable resource.

[Your Name]
[Title], [Organization]

## 5. Campaign Tracking & Management

### Tracking Spreadsheet Schema
| Field | Description |
|---|---|
| Target Page URL | Page where broken link appears |
| Domain | Root domain |
| DR | Domain Rating |
| Traffic | Estimated organic traffic |
| Broken Link URL | Dead link found on their page |
| Wayback URL | Archive snapshot for verification |
| Replacement URL | Client page being proposed as replacement |
| Contact Name | Editor/webmaster name |
| Contact Email | Verified email |
| Date Pitched | Initial send |
| Follow-up 1 | Date |
| Follow-up 2 | Date |
| Status | Prospects / Pitched / Replied / Accepted / Live / Declined / No Response |
| Live Link URL | Page URL once published |
| Anchor Text | Used in live link |
| Do-Follow? | Yes / No |
| Date Confirmed | When link verified live |
| Notes | Any context |

### Prospecting-to-Outreach Pipeline Workflow
- **Daily:** Identify 10–20 new broken link opportunities via Ahrefs / Check My Links
- **2x/week:** Send outreach batches of 20–30 emails
- **Weekly:** Follow up on Day-5 emails; verify any new live links
- **Monthly:** Audit existing live links for removal; add new replacement content assets

### Scaling Broken Link Building
- The highest-leverage play: identify **dead pages with 50+ backlinks** in the niche
- Build a replacement page that's 10x better than the original
- Use Ahrefs → Site Explorer → Enter dead URL → Backlinks → get full list of sites to contact
- This single workflow can generate 30–100 outreach targets from one dead URL

## 6. Success Metrics & KPIs

### Primary KPIs
| Metric | Target |
|---|---|
| Broken Links Found Per Hour (prospecting) | 5–15 qualified opportunities |
| Outreach Response Rate | 15–30% (higher than cold outreach due to value-first hook) |
| Acceptance Rate (of positive responses) | 50–70% |
| Overall Conversion Rate (outreach → live link) | 8–20% |
| Average DR of Links Built | ≥ 35 |
| % Do-Follow | ≥ 85% |
| Links Built Per Month (per client) | 5–15 |

### Secondary KPIs
- Percentage of links from resource/hub pages vs. blog posts
- Average time from pitch to live link (target < 14 days — faster than guest posting)
- Volume of replacement content assets created vs. existing assets leveraged
- DR range distribution of built links

### Broken Link Building vs. Guest Posting Comparison
| Metric | Broken Link Building | Guest Posting |
|---|---|---|
| Response rate | 15–30% | 10–20% |
| Time to link | 1–3 weeks | 3–6 weeks |
| Content creation needed | Sometimes | Always |
| Link quality | Medium–High | Medium–High |
| Scalability | Medium | High |

## 7. What to Avoid

### Ethical & Quality Issues
- **Misrepresenting your content** — Never tell an editor that your page is "the same" as the broken resource if it's fundamentally different. Editors check.
- **Targeting irrelevant broken links** — Just because a link is broken doesn't mean your content is a legitimate replacement. Relevance is non-negotiable.
- **Creating thin replacement content just to qualify** — If you create a page purely as a broken link building vehicle, it won't rank or earn links naturally. Build real, valuable content.
- **Reporting fake broken links** — Do not tell editors a link is "broken" when it's actually redirecting or working. This destroys credibility instantly.

### Outreach Mistakes
- **Not checking the Wayback Machine** — Always verify what the original page was about before suggesting your replacement. Wrong suggestions kill response rates.
- **Over-explaining in the email** — Keep it short and value-first. Long emails about why your content is great read as spam.
- **Sending more than 3 follow-ups** — Beyond 3 touches, you're harassing the editor and damaging future relationship potential.
- **Not verifying the email address** — High bounce rates from unverified emails damage your sending domain's reputation.
- **Contacting the same site more than once per month** — If they don't respond to a broken link campaign, let the domain rest for 60+ days before attempting any new outreach.

### Google Penalty Risks
- **Over-optimized anchor text** — If the editor asks what anchor to use, always suggest branded or descriptive anchors; never keyword-stuffed exact-match text
- **Link exchanges as quid pro quo** — Offering "I'll fix your broken link if you link to me" as an explicit exchange could be interpreted as a link scheme
- **Building too many links too fast** — Even legitimate link building velocity that's unusually high can trigger algorithmic review; keep monthly new referring domains gradual
