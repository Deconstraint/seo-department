# Landing Pages — SEO Playbook

> **Purpose:** Build, optimize, and scale SEO-focused landing pages that rank for high-intent keywords and convert visitors into leads or customers.

---

## When to Use This Playbook

- Client needs pages targeting transactional or commercial-investigation keywords
- Building service pages, location pages, product category pages, or comparison pages
- Existing landing pages are not ranking despite technical health
- Client is targeting competitive, high-value keywords where content depth matters
- Setting up a hub page that funnels into a content cluster
- Launching a new service, product, or market expansion

---

## Step-by-Step Process

### Step 1: Keyword Intent Classification

1. Identify target keyword(s) for the page
2. Classify intent:
   - **Transactional** ("buy X," "hire X," "X pricing") → conversion-focused page
   - **Commercial-investigation** ("best X," "X vs Y," "X for [use case]") → comparison/feature-rich page
   - **Navigational** (branded queries) → brand-reinforced page
3. Validate intent by checking what Google actually ranks: if top 10 are product/service pages, build a landing page. If they're blog posts, don't.
4. Identify primary + 2–4 secondary keywords for this page
5. Check current rankings for these terms (is client already ranking? Where?)

### Step 2: Competitive Gap Analysis

1. Pull top 5 ranking URLs for the primary keyword in Ahrefs/Semrush
2. For each competitor, note:
   - Word count
   - Page structure (H2s, H3s)
   - Trust signals (testimonials, reviews, certifications, case studies)
   - CTAs used
   - Media types (video, interactive tools, comparison tables)
   - Schema markup present
3. Score each competitor on a 1–5 scale for: content depth, UX, trust signals, technical SEO
4. Identify what your page will do BETTER than the current top ranker
5. Note content gaps — topics/sections none of them cover

### Step 3: Page Architecture Planning

1. Define the page URL structure: `/[service]/[location]` or `/[category]/[product]`
2. Keep slugs short, keyword-rich: `/seo-services/local-seo` not `/services/search-engine-optimization-local-services`
3. Map where this page sits in the site hierarchy (parent category → this page → sub-pages)
4. Plan the H1 through H4 heading structure before writing
5. Identify the primary CTA: form, phone call, demo booking, purchase

### Step 4: Content Brief for Landing Page

Complete the **Landing Page Content Brief Template** (see below).

### Step 5: Above-the-Fold (Hero) Optimization

1. H1 must contain primary keyword — write it as a value proposition, not just a label
   - Weak: "SEO Services"
   - Strong: "SEO Services That Drive Measurable Organic Growth"
2. Add a supporting subheading that reinforces the H1 and adds a secondary keyword
3. Include primary CTA button above the fold
4. Hero image/video should be relevant and fast-loading (WebP, < 100KB)
5. Add social proof element above the fold: client count, avg results, star rating

### Step 6: Body Content Structure

1. **Problem/Need Section** (H2): Identify the pain the user has — builds empathy and relevance
2. **Solution Section** (H2): Explain your service/product as the answer — include primary keyword
3. **How It Works** (H2): Step-by-step breakdown of the process — builds trust and reduces friction
4. **Features/Benefits Section** (H2): Bullet format, benefit-focused (not feature-dumping)
5. **Social Proof** (H2): Case studies, testimonials, logos, specific results with numbers
6. **FAQ Section** (H2): Target People Also Ask questions for the keyword — add FAQ schema
7. **Final CTA Section**: Restate offer + urgency/value + form or button

### Step 7: On-Page SEO Execution

1. **Title tag:** `[Primary Keyword] | [Unique Value Prop] | [Brand]` — ≤ 60 chars
2. **Meta description:** ≤ 160 chars, include primary keyword, clear CTA, unique differentiator
3. **URL:** Short, keyword-first, no filler words
4. **H1:** One per page, contains primary keyword, 50–70 chars
5. **H2s:** Contain secondary keywords, descriptive
6. **Image alt text:** Descriptive, keyword-aware (not stuffed)
7. **Internal links:** Link from 3–5 related blog posts/pages; link from this page to 3–5 supporting content pieces
8. **Schema:** Add `LocalBusiness`, `Service`, `Product`, or `FAQPage` schema as applicable
9. **Canonical:** Self-referencing canonical
10. **Page speed:** LCP < 2.5s, CLS < 0.1, FID/INP < 200ms

### Step 8: Conversion Rate Optimization (CRO) Layering

1. Place CTA button in at least 3 locations: hero, mid-page, bottom
2. Reduce form fields to minimum needed (Name + Email + optional note = good)
3. Add trust badges near CTAs: SSL, reviews, guarantees
4. Use directional cues (arrows, eye-lines in photos) pointing toward CTAs
5. A/B test headline and CTA copy after 500+ visitors (use Google Optimize or VWO)

### Step 9: Internal Linking Architecture

1. This page should be linked from:
   - Homepage (if high-priority)
   - Parent category/pillar page
   - 2–3 blog posts on related topics (add links to existing posts)
   - Relevant sidebar or navigation menu
2. This page should link to:
   - 2–3 supporting blog posts (cluster content)
   - Related service/product pages
   - Case studies or testimonials pages
3. Anchor text: use exact-match sparingly (1–2x), partial match and natural language for rest

### Step 10: Launch Checklist

- [ ] All images compressed and have alt text
- [ ] Title tag and meta description set
- [ ] Canonical URL set to self
- [ ] Schema markup validated (Google Rich Results Test)
- [ ] Page added to XML sitemap
- [ ] 301 redirects in place (if replacing an old URL)
- [ ] Indexed in Search Console (submit URL)
- [ ] GSC property verified
- [ ] GA4 conversion event set up for form/CTA
- [ ] Page is mobile-responsive (test on real device)
- [ ] Internal links pointing TO this page are live

### Step 11: Post-Launch Monitoring & Iteration

1. Track keyword rankings weekly for 90 days
2. At 30 days: check GSC impressions/clicks; if no impressions, check indexing
3. At 60 days: if ranking 11–30, add more content depth and semantic coverage
4. At 90 days: if ranking but not converting, run CRO audit
5. Update page quarterly with new stats, testimonials, and content additions

## Content Structure for Humans vs. Search Engines

### For Humans (Conversion-First)
- Lead with the outcome/benefit, not the feature
- Use "you" language throughout ("you'll get," "your results")
- Keep paragraphs ≤ 3 sentences
- Use subheadings as mini-value props (reader should understand the page just from headings)
- Social proof near every CTA
- Reduce cognitive load: no clutter, clear path forward
- Urgency or scarcity where authentic (limited availability, enrollment periods)

### For Search Engines (Rank-First)
- Keyword in title tag, H1, first paragraph, one H2
- LSI/semantic keywords (from NLP tools like Surfer) throughout
- FAQ section targeting PAA snippets
- Schema markup (FAQPage, Service, LocalBusiness)
- Fast page load (Core Web Vitals compliance)
- Internal link equity flowing in from high-authority pages
- Mobile-optimized layout
- Self-referencing canonical

## Word Count Guidelines

| Page Type | Min | Target | Notes |
|---|---|---|---|
| Service page | 800 | 1,500 | Match SERP, include FAQ |
| Location page | 600 | 1,200 | Geo-specific content required |
| Product page | 400 | 800 | UX matters more here |
| Comparison/vs page | 1,200 | 2,000 | Cover all angles |
| Category/hub page | 1,000 | 1,800 | Link to sub-pages |

## Performance Metrics & KPIs

| Metric | Target | Check At |
|---|---|---|
| Impressions (GSC) | Growing within 30 days | 30 days |
| Average position | Top 10 within 6 months | Monthly |
| CTR | ≥ 5% for branded, ≥ 2% for non-branded | Monthly |
| Conversion rate | ≥ 2–5% (varies by industry) | Monthly |
| Bounce rate | ≤ 60% | Monthly |
| Time on page | ≥ 1:30 | Monthly |
| Core Web Vitals | All green | At launch + quarterly |

## Common Mistakes to Avoid

- Targeting informational keywords on a transactional page (intent mismatch)
- Too many CTAs competing with each other
- No internal links pointing TO the landing page (orphaned page)
- Missing FAQ section (missed PAA opportunity)
- No schema markup
- Generic H1 that doesn't include keyword or value prop
- Page speed > 3s (kills both rankings and conversions)
- No conversion tracking set up before launch
