# Blog Content — SEO Playbook

> **Purpose:** Plan, write, and optimize blog content that ranks, earns traffic, and builds authority for any client.

---

## When to Use This Playbook

- Client needs organic traffic growth via informational content
- Building or expanding a blog from scratch
- Targeting mid-funnel "how to" and "what is" queries
- Supporting a content cluster or topical authority strategy
- Creating assets that attract backlinks and social shares
- Client has a content calendar that needs SEO scaffolding

---

## Step-by-Step Process

### Step 1: Keyword Research & Topic Selection

1. Pull seed keywords from client's core services/products
2. Expand via Ahrefs/Semrush "Keyword Ideas" and "Questions" filters
3. Filter for:
   - KD (Keyword Difficulty) ≤ 40 for new sites, ≤ 60 for established
   - Monthly search volume ≥ 100 (lower OK for high-value commercial niches)
   - Informational or commercial-investigation intent
4. Group keywords by topic cluster (parent + related child keywords)
5. Prioritize by: Search volume × relevance × competition gap

### Step 2: SERP Analysis

1. Google the target keyword manually — note:
   - What content types rank (listicles, how-tos, comparisons, guides)
   - Average word count of top 5 results (use SERP word count tools)
   - Presence of featured snippets, People Also Ask, image packs
   - Domain authority of top rankers (are they beatable?)
2. Identify content gaps: what do top results NOT cover that searchers want?
3. Note if Google is serving blog posts or product/landing pages — match format

### Step 3: Content Brief Creation

Fill out the **Content Brief Template** (see below) for every post before writing begins.

### Step 4: Title & H1 Optimization

1. Include primary keyword in title (front-loaded preferred)
2. Keep title under 60 characters to avoid truncation in SERPs
3. Use power words for CTR: "Ultimate," "Complete," "Proven," "Step-by-Step"
4. H1 can mirror title or be a slight variation — keep keyword present
5. A/B test titles using tools like Coschedule Headline Analyzer

### Step 5: Content Outline

1. Map H2s based on PAA questions + top-ranking headings
2. Include target keyword in at least 1 H2
3. Plan H3s for detailed sub-sections
4. Outline word count per section based on competitive average
5. Mark where to include: images, tables, CTAs, internal links

### Step 6: Writing the Content

1. Write intro that: (a) acknowledges the problem, (b) promises the solution, (c) establishes credibility
2. Use the "Inverted Pyramid" — key info first, details second
3. Include primary keyword in first 100 words naturally
4. Maintain 60–80% of paragraphs at 3 sentences or fewer
5. Use numbered lists and bullet points for scannable sections
6. Include data, statistics, or case studies to build E-E-A-T
7. Write a conclusion with a clear CTA (newsletter signup, contact, next article)

### Step 7: On-Page SEO Optimization

1. **Title tag:** Primary keyword + brand (60 chars max)
2. **Meta description:** 150–160 chars, include keyword, add value prop + CTA
3. **URL slug:** Short, keyword-rich, no stop words (e.g., `/blog/content-seo-guide`)
4. **Primary keyword density:** 0.5–1.5% (do not stuff)
5. **LSI/semantic keywords:** Sprinkle throughout H2s and body (use Surfer SEO or NLP tools)
6. **Image alt text:** Descriptive, keyword-relevant for hero and key images
7. **Schema markup:** Add Article or HowTo schema where applicable

### Step 8: Internal Linking

1. Link TO this post from 2–3 existing related posts (add links retroactively)
2. Link FROM this post to 3–5 related internal pages (cluster articles, pillar pages, service pages)
3. Use keyword-rich anchor text (not "click here")
4. Prioritize linking to pages that need authority boosts
5. Add a "Related Articles" section at bottom with 3 links

### Step 9: Publish & Technical Checks

1. Set canonical URL (self-referencing)
2. Add Open Graph tags (title, description, image) for social sharing
3. Test page speed — aim for LCP < 2.5s (compress images, lazy-load)
4. Verify mobile rendering in Search Console or BrowserStack
5. Submit URL to Google Search Console for indexing

### Step 10: Post-Publish Promotion & Link Building

1. Share on client's social channels (LinkedIn, Twitter/X, relevant Facebook groups)
2. Email to subscriber list if available
3. Outreach to 3–5 sites that linked to similar content (Ahrefs → "Who links to competitors")
4. Submit to relevant content aggregators (Reddit, Quora, industry newsletters)
5. Update any internal links from newer posts to point to this article

### Step 11: Performance Monitoring & Optimization

1. Track rankings weekly for target keywords (Ahrefs, Semrush, or Search Console)
2. At 90 days: review impressions, clicks, avg. position in GSC
3. If ranking position 6–20: update content, add more depth, improve semantic coverage
4. If CTR is low despite high impressions: rewrite title/meta description
5. If bounce rate is high: improve intro, add jump links, improve readability

## Content Structure for Humans vs. Search Engines

### For Humans (UX-First)
- Short paragraphs (2–3 sentences max)
- Subheadings every 200–300 words
- Bullet points and numbered lists
- Bold key phrases (not keywords — value statements)
- Visual breaks: images, callout boxes, dividers
- Answer the question in the first 100 words (respect their time)
- Logical flow: Problem → Cause → Solution → Action

### For Search Engines (SEO-First)
- Primary keyword in: title, H1, first 100 words, one H2, meta description, URL
- Semantic/LSI keywords throughout body (NLP optimization via Surfer or Clearscope)
- Structured data (Article schema, FAQ schema, HowTo schema)
- Internal link graph connectivity
- Crawlable URL structure
- Fast load time (Core Web Vitals)
- Mobile-first content layout

## Word Count Guidelines by Content Type

| Content Type | Min Words | Target Words | Notes |
|---|---|---|---|
| News/Update Post | 500 | 800 | Timeliness > depth |
| How-To Guide | 1,200 | 1,800 | Match SERP avg |
| Ultimate/Pillar Guide | 2,500 | 4,000+ | Comprehensive coverage |
| Listicle | 1,000 | 1,500 | Per-item depth matters |
| Comparison Post | 1,500 | 2,500 | Cover all angles |
| FAQ / Definition | 600 | 1,000 | Concise, structured |

## Common Mistakes to Avoid

- Writing for assumed intent without validating SERP
- Targeting keywords with KD above site authority
- Publishing without meta description
- Ignoring internal linking (leaving content as orphan pages)
- Not adding schema markup
- Word-stuffing instead of semantic depth
- Publishing and abandoning (no refresh cycle)
